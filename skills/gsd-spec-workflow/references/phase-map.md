# Phase Map

Use this mapping to keep sources of truth distinct.

| Phase | OpenSpec responsibility | GSD responsibility | Superpowers responsibility |
| --- | --- | --- | --- |
| Discuss | Problem, scope, non-goals | Clarify phase objective | Challenge ambiguity |
| Plan | Proposal, specs, design, tasks | Phase plan and order | TDD and verification approach |
| Execute | Acceptance criteria | Task state and evidence | TDD, debugging, reviews |
| Verify | Spec compliance | Phase verification | Tests and completion checks |
| Ship | Archive readiness | PR or release decision | Branch finishing discipline |

OpenSpec tasks are requirements tasks. Convert them into TDD execution tasks
before coding.
